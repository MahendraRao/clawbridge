# ClawBridge Roadmap

## Vision

ClawBridge is evolving into an operational UX and orchestration layer for AI agents.

The goal is to simplify:
- onboarding
- provider orchestration
- diagnostics
- operational reliability
- local AI workflows
- developer experience

---

# Phase 1 — Install + Verify
- frontend scaffold
- backend local API
- OpenClaw detection
- Node/Git verification
- install recommendation flow

# Phase 2 — Provider orchestration
- OpenAI support
- Anthropic support
- Ollama support
- local/custom models
- provider-specific env generation
- secure local config generation

# Phase 3 — Operational UX
- launch orchestration
- provider reachability checks
- activity logs
- richer diagnostics
- troubleshooting workflows

# Phase 4 — Agent Operations
- session visibility
- execution tracing
- provider health metrics
- token usage visibility
- agent observability

# Phase 5 — DevOps + Infrastructure
- Docker integration
- Kubernetes integration
- OpenTelemetry support
- gateway health probes
- deployment diagnostics

# Phase 6 — Platform evolution
- plugin architecture
- packaged desktop app
- SDK extraction
- AI-assisted operational workflows
